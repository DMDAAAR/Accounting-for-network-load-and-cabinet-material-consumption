# Техническая документация проекта: Система учёта сетевой инфраструктуры и расхода материалов (Кабинет 319Б)

Данный документ представляет собой простое и понятное описание учебного веб-приложения для фиксации сетевых точек, учёта дефектов оборудования (с возможностью загрузки фото) и контроля расхода материалов. Документация предназначена для команды разработчиков (студентов) и преподавателя.

---

## 🚀 1. Стек технологий

Проект написан на простом и понятном стеке технологий без использования сложных фреймворков:

- **Язык программирования:** PHP 7.4+.
- **База данных:** MySQL (подключение через безопасный интерфейс PDO).
- **Фронтенд (интерфейс):** HTML5, CSS3, библиотека Bootstrap 5.3.3.

---

## 🏗️ 2. Структура проекта (Файловый MVC)

Код приложения разделен по принципу **Model-View-Controller (Модель-Представление-Контроллер)**. Обработка запросов и внешний вид разделены по разным папкам:

*   **`index.php`** — Главная страница приложения (дашборд).
*   **`db/`** — Папка для подключения к базе данных.
    *   `connectDB.php` — Настройки подключения к MySQL.
*   **`models/`** — Модели (здесь хранятся файлы с SQL-запросами к базе данных):
    *   `dashboard.model.php` — Запросы для главной страницы и статистики.
    *   `defects.model.php` — Запросы для работы с дефектами (добавление, удаление, починка).
    *   `inventory.model.php` — Запросы для учёта сетевых портов/точек.
    *   `materials.model.php` — Запросы для учёта материалов на складе.
    *   `user.model.php` — Запросы для регистрации и входа пользователей.
    *   `logs.model.php` — Запросы для записи действий пользователей в журнал.
*   **`controllers/`** — Контроллеры (обрабатывают действия пользователя, проверяют формы и связывают модели с визуальной частью):
    *   `inventory.controller.php` — Логика добавления и изменения оборудования.
    *   `defects.controller.php` — Логика загрузки фото, редактирования и удаления дефектов.
    *   `materials.controller.php` — Логика управления остатками на складе.
    *   `useMaterial.controller.php` — Логика быстрого списания материалов.
    *   `login.controller.php` / `register.controller.php` / `logout.controller.php` — Логика входа, регистрации и выхода из системы.
*   **`views/`** — Представления (HTML-шаблоны страниц, которые видит пользователь):
    *   `components/` — Повторяющиеся части страниц (`header.view.php` — шапка сайта и меню, `footer.view.php` — подвал сайта).
    *   `index.view.php` — Главная страница со списком сетевых точек.
    *   `inventory.view.php` — Страница инвентаризации (добавление/изменение оборудования).
    *   `defects.view.php` — Журнал дефектов.
    *   `fix_defect.view.php` — Форма списания материалов при починке дефекта.
    *   `materials.view.php` — Страница склада материалов.
    *   `login.view.php` / `register.view.php` — Формы входа и регистрации.
*   **`uploads/`** — Папка, куда физически сохраняются загруженные фотографии дефектов (в подпапку `defects/`).

## 🗄️ 3. База данных (краткое описание таблиц)

База данных состоит из 7 простых таблиц, связанных между собой:

1.  **`users`** — Пользователи системы (хранит логины и зашифрованные пароли).
2.  **`locations`** — Аудитории и кабинеты (например, Кабинет 319Б, Кабинет 335).
3.  **`network_points`** — Точки сети (розетки, коммутаторы, кабели, патч-корды).
4.  **`defects`** — Обнаруженные поломки (название, описание, к какой сетевой точке относится, статус и путь к фотографии).
5.  **`materials`** — Склад материалов (названия, типы вроде кабеля или коннектора, единицы измерения и сколько осталось на складе).
6.  **`material_usage`** — Журнал использования материалов (какой материал, сколько штук/метров, кто списал и при ремонте какого дефекта).
7.  **`logs`** — История действий (кто из пользователей когда зашел, что удалил или добавил).

## 👥 4. Роли пользователей

В системе есть две роли пользователей (записываются в сессию при входе):
*   **Оператор (`operator`):** может просматривать оборудование, добавлять новые точки сети, создавать дефекты, закрывать их при починке со списанием материалов, а также управлять складом.
*   **Администратор (`admin`):** имеет все права оператора, а также может видеть историю действий (логи) всех пользователей на главной странице.

## 🛠️ 5. Как работают основные функции приложения

### 5.1. Авторизация
Пользователь регистрируется через форму регистрации, его пароль хэшируется функцией `password_hash()` и записывается в базу. При входе система проверяет логин и пароль. Если всё верно, данные пользователя записываются в сессию `$_SESSION['user']`, и он получает доступ к закрытым разделам.

### 5.2. Учёт оборудования
На странице «Оборудование» операторы могут добавлять новые розетки, коммутаторы или кабели, привязывая их к конкретным кабинетам. Информацию о любой точке сети можно изменить или удалить.

### 5.3. Журнал поломок (Дефекты) и загрузка фото
Когда на сети что-то ломается, оператор создаёт запись в журнале дефектов. Он может прикрепить фото:
- Фотографии загружаются в папку `uploads/defects/` с уникальным именем.
- Система проверяет, чтобы файл был именно картинкой (JPEG, PNG, GIF, WEBP) и весил не более 2 МБ.
- При удалении поломки или её изменении старые файлы автоматически удаляются с сервера, чтобы не занимать место.

### 5.4. Починка дефектов со списанием материалов
При нажатии кнопки «Починить» открывается форма ремонта. Оператор может выбрать один или несколько материалов со склада (например, 10 метров кабеля и 2 коннектора), которые он потратил на ремонт. После отправки формы:
1.  Указанное количество материалов вычитается со склада (таблица `materials`).
2.  В таблицу `material_usage` записывается информация о расходе.
3.  Дефект закрывается (статус меняется на `closed`).
